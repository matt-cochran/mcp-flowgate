---
title: "Installation"
description: "Install mcp-flowgate from a pre-built binary, Cargo, or Docker."
---

Pick whichever method fits your setup. They all give you the same `mcp-flowgate` binary.

## Pre-built binary (recommended)

Download the latest release for your platform from [GitHub Releases](https://github.com/matt-cochran/mcp-flowgate/releases). Each release includes a `checksums.sha256` file so you can verify the download.

```bash
# Linux x86_64 example
curl -LO https://github.com/matt-cochran/mcp-flowgate/releases/latest/download/mcp-flowgate-x86_64-unknown-linux-gnu.tar.gz
curl -LO https://github.com/matt-cochran/mcp-flowgate/releases/latest/download/checksums.sha256

# Verify
sha256sum -c checksums.sha256 --ignore-missing

# Extract and run
tar xzf mcp-flowgate-*.tar.gz
./mcp-flowgate --help
```

### Available targets

| Platform | Target |
|----------|--------|
| Linux x86_64 | `x86_64-unknown-linux-gnu` |
| Linux ARM64 | `aarch64-unknown-linux-gnu` |
| macOS Intel | `x86_64-apple-darwin` |
| macOS Apple Silicon | `aarch64-apple-darwin` |
| Windows x86_64 | `x86_64-pc-windows-msvc` |

Move the binary somewhere on your `PATH` (e.g., `/usr/local/bin`) and you're set.

## Cargo

If you have a Rust toolchain installed:

```bash
cargo install mcp-flowgate
```

This builds from source, so it takes a couple of minutes. The binary lands in `~/.cargo/bin/`, which is usually already on your `PATH`.

## Docker

```bash
docker run -v $(pwd)/gateway.yaml:/config/gateway.yaml ghcr.io/matt-cochran/mcp-flowgate
```

Mount your config file into `/config/gateway.yaml` and the container handles the rest. This works well for server deployments or if you want to try mcp-flowgate without installing anything locally.

## Verify the installation

Whichever method you used, confirm it's working:

```bash
mcp-flowgate --help
```

You should see the CLI help with the `serve` subcommand listed. Head to the [quick start](/docs/quick-start) to wire up your first capability.

## Optional: install the TUI

The `mcp-flowgate-tui` crate ships a terminal client that drives the gateway interactively. Useful for trying workflows without a real MCP host attached, and for the per-state sub-agent routing (`delegate:`) pattern.

```bash
cargo install mcp-flowgate-tui
```

You get two binaries with the same behavior — `flowgate` (the short, recommended invocation) and `flowgate-tui` (the long form for environments where `flowgate` collides with something else). The TUI auto-discovers the `mcp-flowgate` binary; if you've put it somewhere unusual, point at it with `MCP_FLOWGATE_PATH=/path/to/mcp-flowgate`.

See [The TUI](/guides/tui/) for the rundown — picking a workflow, driving transitions, routing cognitive states to different agent configs, and reading the logs the TUI drops in `~/.cache/flowgate/logs/`.

## Wire flowgate into your editor

With the binaries installed, generate the editor config files in one shot:

```bash
cd your-project
flowgate mcp init --cursor --claude-desktop
```

That writes `.mcp.json` (Claude Code, Continue, Cline), `.cursor/mcp.json`, and a `claude_desktop_config.json` snippet. Zed needs a slightly different setup — copy the snippet from the guide.

See [Wire flowgate into your editor](/guides/editors/) for the full per-editor walkthrough, both the MCP path (every editor) and the ACP path (Zed today, others adopting).
